# Implementation Plan: Shops Onboarding — New 4-Step Flow Redesign

**Branch**: `035-shops-onboarding-redesign` | **Date**: 2026-05-17 | **Spec**: [spec.md](./spec.md)

## Summary

Redesign the shops onboarding from a 4-step flow (Business → Sales → Setup → Review) into a new 4-step flow (Business+Sales merged → Store Setup → First Products → Review & Launch). Add mode-aware dashboard rendering so the dashboard adapts its layout, stats, banner, and sidebar based on the `shops_mode` selected during onboarding. All changes are scoped to `apps/shops-app` with no new packages.

## Technical Context

**Language/Version**: TypeScript 5.x, strict mode

**Constraints**: No new npm packages; backward-compatible session storage keys; `pnpm --filter shops-app build` must pass  
**Scale/Scope**: Single app feature (~12 component files changed/created)

## Architecture Decisions (summary — full rationale in research.md)

## Key Implementation Notes

### Step 1: Business Type Grid (3×3 + Other)
- Use emoji characters directly in JSX (no Icon component needed per spec assumption)
- Grid: 3 columns on desktop, 2 on tablet, 1 on mobile (sm:grid-cols-2 lg:grid-cols-3)
- "Other" tile reveals `<input type="text">` below the grid when selected
- "Continue →" disabled until: `businessType !== null && (businessType !== "other" || customText.trim() !== "") && salesModel !== null`

### Step 1: Sales Model Cards
- Three cards stacked vertically (or 3-col on larger screens as specified)
- Physical: emerald tint, building-2 icon, "IN-STORE" tag
- Online: blue tint, globe icon, "E-COMMERCE" tag  
- Both: purple tint, zap icon, "RECOMMENDED" tag, default selected, full-width prominent border with glow
- Features list inside each card

### Step 1: Module Preview
- Renders below the sales model cards
- Updates immediately on `salesModel` state change
- Uses `Badge` component from `@nexoraxs/ui` for module pills
- Physical modules: POS (cyan), Inventory (blue), Branches (purple), Reports (gray)
- Online modules: Storefront (blue), Orders (cyan), Checkout (emerald), Reports (gray)
- Both: all 6 + "+3 more" badge

### Step 2: Store Setup
- Country dropdown drives auto-fill of Currency + Timezone using `COUNTRY_DEFAULTS`
- User can override either after auto-fill
- Live preview card sticky on desktop right column
- Add `branchAddress` optional text field below branch name
- Add `branchTimezone` select below currency (or text display auto-filled from country)

### Step 3: Products
- Single product row rendered by default (not empty — show one empty form)
- "+ Add another product" button: hidden when `products.length >= 3`
- × button per row: removes that index from array
- Price field prefix shows currency from step 2 state (`storeSetup.branchCurrency`)
- "Skip for now →" link: sets products to `[]` and advances step

### Step 4: Review & Launch  
- Left column: Workspace (read-only) · Business Type (with custom label if "other") · Sales Model
- Right column: Store + Branch · Country · Currency · Timezone · Products count
- Full-width modules grid using `ModuleDefinition` from data-model
- Each module as a card: green checkmark icon if enabled, gray × if disabled + reason text
- "Launch Store →" button: calls `completeOnboarding()` then `router.push("/dashboard")`
- Remove old "Next recommended actions" sidebar — replaced by modules grid

### Dashboard Mode-Aware Content
- Physical mode: emerald badge, POS stats set, cyan POS banner, POS quick actions
- Online mode: blue badge, online stats set, blue Storefront banner, e-commerce quick actions
- Both mode: purple badge, all 4 stats, both banners, all quick actions (existing behavior preserved)
- Stats are defined as three arrays; selected by mode switch
- Quick actions similarly defined per mode

### Sidebar Mode-Awareness
- `Sidebar` is "use client" — reads `getMode()` via `useSyncExternalStore`
- Add Storefront nav item (currently missing) shown only for "online" and "both"
- Hide POS nav item for "online" mode
- Storefront item: `{ label: "Storefront", href: "#", icon: "shopping-bag", disabled: true }`
