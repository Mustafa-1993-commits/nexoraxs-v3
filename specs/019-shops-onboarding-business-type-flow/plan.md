# Implementation Plan: Shops Onboarding Business Type Flow

**Branch**: `019-shops-onboarding-business-type-flow` | **Date**: 2026-05-14 | **Spec**: [spec.md](./spec.md)

## Summary

Replace the current 3-step Shops onboarding with a 4-step flow: (1) Business Type, (2) Sales Model, (3) Store Setup, (4) Review. Migrate `ShopsMode` type values. Add 7 new business-type icons to `Icon.tsx`. Update 3 dashboard components that depend on the old mode values. Create 4 new step-panel components. Fully rewrite the onboarding page.

### 7. `app/onboarding/page.tsx` — Full Rewrite

**State**:
```typescript
const mounted = useSyncExternalStore(...)
const [currentStep, setCurrentStep] = useState<1|2|3|4>(1)
const [businessType, setBusinessType] = useState<BusinessType | null>(null)
const [salesModel, setSalesModel] = useState<ShopsMode | null>(null)
const [storeSetup, setStoreSetup] = useState<StoreSetupData>({
  storeName: "", branch: "", currency: "EGP", country: "Egypt"
})
const isComplete = mounted ? isOnboardingComplete() : false
```

**Lazy pre-population** (on mount):
- If `getBusinessType()` is non-null, pre-set `businessType` 
- If `getMode()` is non-null, pre-set `salesModel`
- Pre-populate `storeSetup` from `getStoreName()`, `getBranch()`, `getCurrency()`, `getCountry()`

**canProceed logic**:
- Step 1: `businessType !== null`
- Step 2: `salesModel !== null`
- Step 3: `storeSetup.storeName.trim() !== "" && storeSetup.branch.trim() !== ""`
- Step 4: `true` (always can proceed — shows "Finish setup")

**handleContinue**:
- Step 1: no persistence yet (persisted on step 3 + finish)
- Step 2: no persistence yet
- Step 3: writes `setStoreName`, `setBranch`, `setCurrency`, `setCountry` to sessionStorage
- Step 4 ("Finish setup"): writes `setBusinessType(businessType!)`, `setMode(salesModel!)`, `completeOnboarding()`, then `router.push("/dashboard")`

**Navigation**:
- `handleBack()`: decrements `currentStep`
- `handleGoToStep(n)`: sets `currentStep` to `n`

**Layout**:
- Header (unchanged brand bar from old onboarding)
- `{!mounted → skeleton}`
- `{isComplete → CompletionState (existing component)}`
- `{else → step flow}`

**Step flow**:
- `StepIndicator` (4 steps: Business/Sales/Setup/Review)
- Conditional renders for steps 1–4
- Fixed bottom nav bar: Back (hidden on step 1) + Continue/Finish setup
