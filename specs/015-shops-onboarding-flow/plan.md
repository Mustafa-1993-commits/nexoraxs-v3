## Research Summary

See [research.md](./research.md) for full decision log. Key choices:

| Decision | Choice | Key Reason |
|----------|--------|-----------|
| Step state | `useState<1\|2\|3>` — not persisted | Steps are transient; only outcomes (mode, done flag) persist |
| SSR safety | Lazy `useState(() => ...)` initializer | Avoids lint violations and SSR crashes; matches existing pattern |
| lib/mode.ts | Add 3 exports with `shops_onboarding_done` key | Mirrors existing `shops_mode` pattern; single file for all session state |
| Step 2/3 content | Rendered inline in OnboardingPage | Avoids wrong directional dependency on dashboard components |
| Dashboard guard | `mode === null` → prompt card; `mode && !done` → soft banner | Hard block for missing mode; soft nudge for incomplete setup |

## Detailed Implementation Notes

### lib/mode.ts — New Exports

Three new exports follow the identical window-guard pattern as existing helpers:

- `completeOnboarding(): void` — writes `"true"` to `shops_onboarding_done` in sessionStorage
- `isOnboardingComplete(): boolean` — returns `sessionStorage.getItem("shops_onboarding_done") === "true"`
- `resetOnboarding(): void` — removes `shops_onboarding_done` from sessionStorage

### app/onboarding/page.tsx — State Shape

```
isComplete: boolean        — lazy init: isOnboardingComplete()
currentStep: 1 | 2 | 3    — init: 1 (always, not persisted)
selected: ShopsMode | null — lazy init: getMode() (restores prior selection on refresh)
```

**Render branches:**
- `isComplete === true` → completion state: headline "You're all set", body copy, `<a href="/dashboard">` CTA
- `isComplete === false` → multi-step flow

**Multi-step flow layout:**
- Fixed header (brand bar, unchanged)
- Step indicator: 3 numbered pills with connecting lines — active pill filled, inactive hollow
- Step panels (only one visible at a time):
  - Step 1: `ModeCard` list (imported from `@/components/onboarding/ModeCard`) + "Next" button (disabled when `!selected`)
  - Step 2: Inline store profile (mock values: "Mustafa's Co.", "Maadi Main", "EGP", mode label) + Back / Next buttons
  - Step 3: Inline checklist (5 items, all unchecked) + Back button + "Finish setup" button

**Handlers:**
- `handleNext()`: Step 1 only — calls `setMode(selected)` before advancing; all steps advance `currentStep`
- `handleBack()`: decrements `currentStep`
- `handleFinish()`: calls `completeOnboarding()`, then `router.push("/dashboard")`

**Mode label map (used in Step 2 display):**
- `"business"` → "Business Management"
- `"store"` → "Storefront"
- `"both"` → "Both"

### app/(app)/dashboard/page.tsx — Guard Logic

Add two lazy useState values at the top of `DashboardPage`:

```typescript
const [mode] = useState<ShopsMode | null>(
  () => typeof window === "undefined" ? null : getMode()
);
const [onboardingDone] = useState<boolean>(
  () => typeof window === "undefined" ? false : isOnboardingComplete()
);
```

**Render logic:**
- `mode === null` → render only the setup prompt card; all operations content hidden
- `mode !== null && !onboardingDone` → render full operations content + subtle "finish setup" banner above header
- `mode !== null && onboardingDone` → render full operations content; no banner

**Setup prompt card:**
- Chip: `{"// getting started"}`
- Heading: "Complete your setup to get started"
- Body: "Select your shop mode and finish the setup flow to unlock your dashboard."
- CTA: `<a href="/onboarding" className="btn-primary …">Start setup →</a>`

**Soft banner (low priority, shown when mode exists but `!onboardingDone`):**
- Single slim bar above the main content
- Text: `Setup not complete · ` + `<a href="/onboarding" className="…">Finish setup</a>`
- Visual weight: border only, `bg-amber-500/5 border-amber-500/15`

### Step Indicator

Three numbered pills in a horizontal row with connecting lines. Active step: solid blue fill. Completed steps (index < currentStep): checkmark or muted fill. Upcoming: hollow ring. All implemented with inline TailwindCSS — no new component file.

### Checklist Items — Step 3 (inline)

Five items, identical labels to `SetupChecklist.tsx`, but rendered directly in the onboarding page:
1. Choose shop mode
2. Add business profile
3. Add first product
4. Invite team member
5. Review settings

Each item: `<span className="h-4 w-4 rounded-full border-2 border-white/20 flex-shrink-0" />` + label.
