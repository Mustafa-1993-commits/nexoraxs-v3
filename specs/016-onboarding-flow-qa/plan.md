## Confirmed Fixes

### Fix 1 — Per-Step Heading in `app/onboarding/page.tsx`

**Problem**: The `StepFlow` component renders a static `<h1>Choose your shop mode</h1>` and subtitle regardless of `currentStep`. Users on Step 2 and Step 3 see the wrong heading.

**Fix**: Define a `stepContent` map keyed by `OnboardingStep` and render the active entry:

```typescript
const stepContent: Record<OnboardingStep, { heading: string; subtitle: string }> = {
  1: {
    heading: "Choose your shop mode",
    subtitle: "Select how you want to use NexoraXS Shops. You can change this later in Settings.",
  },
  2: {
    heading: "Set up your store",
    subtitle: "Choose a branch and default currency. You can update these later in Settings.",
  },
  3: {
    heading: "Review your setup",
    subtitle: "Confirm the details below before entering your dashboard.",
  },
};
```

Replace the hardcoded `<h1>` and `<p>` in `StepFlow` with `{stepContent[currentStep].heading}` and `{stepContent[currentStep].subtitle}`.

**Scope**: `StepFlow` component only. No other component changes.

---

### Fix 2 — Dynamic Branch in `components/dashboard/Topbar.tsx`

**Problem**: The Topbar branch selector shows `"Maadi Main"` hardcoded. After onboarding with "Nasr City", the StoreProfile card correctly shows "Nasr City" but the Topbar still shows "Maadi Main" — a confirmed UX inconsistency.

**Fix**: Add the `useSyncExternalStore` pattern to Topbar (same as `StoreProfile.tsx`) and read `getBranch()` with a `"Maadi Main"` fallback.

```typescript
import { useSyncExternalStore } from "react";
import { getBranch } from "@/lib/mode";

// Inside Topbar():
const mounted = useSyncExternalStore(() => () => {}, () => true, () => false);
const branch = mounted ? (getBranch() ?? "Maadi Main") : "Maadi Main";
```

Replace the hardcoded `"Maadi Main"` string in the branch selector JSX with `{branch}`.

**Scope**: Branch selector text only. No other Topbar changes.
