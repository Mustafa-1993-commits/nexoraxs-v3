# Implementation Plan: Shops Onboarding V2

**Branch**: `014-shops-onboarding-v2` | **Date**: 2026-05-13 | **Spec**: [spec.md](./spec.md)

## Summary

Upgrade the Shops App onboarding and dashboard with: a richer mode-selection screen (icon-based cards with feature bullets and a "Recommended" badge), a static setup checklist, a store profile card, and mode-adapted next steps on the dashboard. All changes are UI-only within `apps/shops-app`. No new packages. All inter-app or external links use `href="#"`.

## Research

### Icon mapping (Icon.tsx audit)

The existing `IconName` union in `components/ui/Icon.tsx` contains 28 icons. The following are assigned to mode cards:

| Mode | Icon | Rationale |
|------|------|-----------|
| Business Management | `"chart-bar"` | Conveys operations/reporting |
| Storefront | `"shopping-bag"` | Conveys commerce/storefront |
| Both | `"dashboard"` | Conveys unified/all-in-one view |

For the SetupChecklist items, the checklist icon will be an inline SVG circle (no new icon needed — rendered directly in JSX).

For the NextSteps card, all action items use existing icons: `"chart-bar"` (reports), `"users"` (invite team), `"settings"` (configure), `"package"` (add product), `"scan-line"` (POS).

### Mode-adapted next steps content

| Mode | Steps (label → icon) |
|------|----------------------|
| `"business"` | "Review reports" → `"chart-bar"` · "Invite team member" → `"users"` · "Configure tax settings" → `"settings"` |
| `"store"` | "Add first product" → `"package"` · "Configure POS" → `"scan-line"` · "Set up inventory" → `"package-search"` |
| `"both"` / `null` | "Add first product" → `"package"` · "Review reports" → `"chart-bar"` · "Configure POS" → `"scan-line"` |

### Mode label display mapping

| Raw value | Display label |
|-----------|---------------|

### `"use client"` scope

- `ModeCard.tsx` — already `"use client"` (button interaction)
- `app/onboarding/page.tsx` — already `"use client"` (useState + router)
- `app/(app)/dashboard/page.tsx` — already `"use client"` (useState for period)
- `SetupChecklist.tsx` — Server Component (no state, no browser APIs)
- `StoreProfile.tsx` — `"use client"` (reads getMode() via lazy useState)
- `NextSteps.tsx` — `"use client"` (reads getMode() via lazy useState)

### Lazy useState pattern (mandatory for getMode reads)

```typescript
// Correct — guards against SSR, avoids react-hooks/set-state-in-effect lint error
const [mode, setMode] = useState<ShopsMode | null>(
  () => typeof window === "undefined" ? null : getMode()
);
```
