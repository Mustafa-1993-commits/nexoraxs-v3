### Page 5 — Settings (`app/(app)/settings/page.tsx`)

**Component type**: Client Component (`"use client"`)
**Hydration pattern**: `useSyncExternalStore(() => () => {}, () => true, () => false)` for mount detection

**Sections** (4 × `.card` blocks):

1. **Store Profile** — reads `getBranch() ?? "Maadi Main"` and `getCurrency() ?? "EGP"` after mount; displays store name "Mustafa's Co." (static). All read-only rows using `DetailRow`-style layout (icon + label). Disabled "Edit" chip.

2. **Shop Mode** — reads `getMode()` after mount; maps to label ("Business Management" / "Storefront" / "Both" — default "Both"). Displays as a read-only badge row.

3. **Team** — 2 static team member rows: "Mustafa Mohamed · Owner · Active" and "Placeholder Staff · Cashier · Active". Disabled "Invite" button chip.

4. **Advanced** — 2 read-only toggle rows:
   - "Auto-close orders" — visual toggle OFF (static span)
   - "Email receipts" — visual toggle ON (static span, blue)

**"Save changes" button**: Rendered as `<button type="button" disabled>` with `opacity-40 cursor-not-allowed` classes.

**Loading skeleton**: While `!mounted`, show a simple skeleton card instead of the session-dependent content (prevents flash).
