# UI/UX Guidelines

Act like a product-minded designer-engineer.

## General UI Rules

Prefer:
- simple flows
- clear hierarchy
- clean spacing
- readable typography
- obvious primary actions
- useful empty states
- clear loading states
- clear error states
- responsive layouts
- accessible contrast
- consistent buttons/cards/forms

Avoid:
- clutter
- too many actions per screen
- complex UI before real features exist
- mixing unrelated modules
- dashboards full of fake features
- confusing onboarding

## SaaS Style

If no design system exists, use a minimal SaaS style:
- cards
- soft borders
- generous spacing
- clear section titles
- simple navigation
- readable body text
- calm neutral colors
- obvious call-to-action buttons

## Onboarding

Onboarding should be:
- short
- obvious
- reversible later if settings allow
- focused on one decision at a time

For shops mode selection:
- Business Management
- Online Store
- Both

Explain each option briefly.

Do not add product/inventory/order UI during foundation unless explicitly requested.

## Dashboards

A foundation dashboard should show:
- app name
- workspace
- selected mode/status
- next step placeholder
- no fake heavy modules

## core-app UI

Should feel like:
- secure entry point
- workspace selector
- app launcher
- account/subscription shell

Should not feel like:
- shop management system
- admin console
- inventory system

## admin-app UI

Should feel like:
- internal SaaS control room
- operational admin panel
- clear tables and filters
- user/workspace/subscription management

Should not feel like:
- customer business app

## shops-app UI

Should feel like:
- app-specific workspace
- shop/business management environment
- store management environment

But during foundation:
- onboarding + placeholder only
